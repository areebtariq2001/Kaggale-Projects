# Movie Review Sentiment Analysis — NLP Project

## Overview
This project uses Natural Language Processing (NLP) to classify movie reviews as **positive** or **negative** based purely on their text content. Unlike the other projects in this portfolio (which used structured/tabular data), this project works directly with raw, unstructured text — a core NLP skill.

## Dataset
**IMDB Dataset of 50K Movie Reviews** — a large, perfectly balanced dataset of movie reviews labeled as positive or negative.

Source: [Kaggle — IMDB Dataset of 50K Movie Reviews](https://www.kaggle.com/datasets/lakshmi25npathi/imdb-dataset-of-50k-movie-reviews)

| Column | Description |
|---|---|
| review | Raw text of the movie review |
| sentiment | Target label — positive or negative |

## Project Workflow
1. **Data Cleaning** — Removed HTML tags (e.g. `<br />`), special characters, and numbers; converted text to lowercase; stripped extra whitespace.
2. **Exploratory Data Analysis (EDA)** — Checked sentiment balance, examined review length distribution by sentiment.
3. **Text Visualization** — Generated WordClouds for positive and negative reviews to visually identify the most common words in each class.
4. **Feature Engineering** — Converted cleaned text into numeric features using **TF-IDF Vectorization** (5,000 most informative words).
5. **Modeling** — Trained and compared two classification models:
   - Logistic Regression
   - Multinomial Naive Bayes (a classic, fast algorithm for text classification)
6. **Evaluation** — Compared models using accuracy, precision, recall, F1-score, and confusion matrices.
7. **Custom Prediction Demo** — Built a `predict_sentiment()` function to test the model on new, unseen review text.

## Key Findings
- WordCloud visualizations clearly show that positive reviews are dominated by words like *great*, *best*, *love*, while negative reviews feature words like *bad*, *worst*, *boring*.
- TF-IDF + Logistic Regression is a simple yet powerful baseline for text classification tasks.
- Naive Bayes, despite its simplicity, performs competitively on this dataset — a useful reminder that simpler models can match more complex ones on well-structured text features.
